## Files
---------------------------------------------------------------------------
Piece.java : This class represents a piece in the game.

Cell.java: This class represents a cell in the game board.

QuoCell.java: This class represents a cell in the Quoridor board. It extends Cell.

Board.java: This abstract class represents a generic game board.

TTTBoard.java: This class represents a board in the game Tic Tac Toe. It extends Board.

QuoridorBoard.java: This class represents a board in the game Quoridor.

Player.java: This class represents a player in the game.

QuoPlayer.java: This class represents a player in the game Quoridor. It extends Player.

BoardGame.java: This abstract class represents a generic board game.

TicTacToe.java: This class represents the game TicTacToe extending BoardGame. It contains all the basics of interaction and rules of the game.

OrderAndChaos.java: This class represents the Order and Chaos game. It extends TicTacToe because the game is a variation of Tic Tac Toe.

Quoridor.java: This class represents the game Quoridor. It extends BoardGame.

GameLauncher.java: This class is responsible for menu and start of the three sub boardgames.

Main.java: This class is the main class to start the board game.



## Notes
---------------------------------------------------------------------------
1. In the Quoridor game, each player is highlighted by a unique color in the board display. The pieces and fences are all displayed in the corresponding player's color. We think this feature can help players to distinguish the pieces and fences of different players more intuitively.
2. In the Quoridor game, when a player choose to make a move, the program will automatically calculate all valid moves for the player and mark them with purple color in the board display. We think this feature can help players to make decisions more easily, without having to calculate the valid moves by themselves.
3. In the Quoridor game, players can choose the amount of players (2 or 4) to play the game. When there are 4 players.
4. In the Quoridor game, the way to place a fence is to input the numbers of 2 cells where the fence is placed and on which side is the fence placed. For example, if a player wants to place a fence between cell 1 and cell 2 on the down side, the player should simply input "1 2 D". The program will automatically check if the fence is placed in a valid position and if the fence blocks any player's path to the destination. We think this feature and input instructions can help players place fences more intuitively and easily.
5. When placing a fence, if there is piece in the cells and player wants to put a fence near the piece, the program will allow player to just type the piece symbol and side, without the need to recognize the cell numbers. 
6. When player thinks he made a wrong option, he can just type 0 to go back to the upper option. For example, when choosing fences player can type 0 to go back to the upper option to choose move or fences.
